# Post Semantic Search

This document describes the post embedding index added for `DysonNetwork.Sphere` and the embedding gRPC surface exposed from `DysonNetwork.Insight`.

## Summary

Posts can now be searched with either:

- default semantic vector search
- full-text search via `?searchEngine=fulltext`, `?searchEngine=full-text`, or `?searchEngine=full_text`

The default search engine is configurable in `DysonNetwork.Sphere/appsettings.json`:

```json
"Posts": {
  "SearchEngineDefault": "semantic"
}
```

The semantic path uses an embedding generated by `DysonNetwork.Insight`, stores a post-level vector index in `DysonNetwork.Sphere`, and ranks matching posts by cosine distance.

## Components

### Insight

Insight exposes a dedicated embedding gRPC service:

- proto: `../DysonSpec/proto/embedding.proto`
- server: `DysonNetwork.Insight/Thought/Memory/EmbeddingGrpcService.cs`

Service:

```proto
service DyEmbeddingService {
  rpc GenerateEmbedding(DyGenerateEmbeddingRequest) returns (DyGenerateEmbeddingResponse);
  rpc GenerateEmbeddings(DyGenerateEmbeddingsRequest) returns (DyGenerateEmbeddingsResponse);
}
```

This service wraps the existing `EmbeddingService` already used inside Insight.

### Sphere

Sphere stores the embedding index locally in a new table backed by pgvector.

Model:

- `DysonNetwork.Shared/Models/PostIndex.cs`

Database wiring:

- `DysonNetwork.Sphere/AppDatabase.cs`
- migration: `DysonNetwork.Sphere/Migrations/20260511120000_AddPostIndices.cs`

Stored fields:

- `post_id`
- `source_text`
- `source_hash`
- `embedding`
- `indexed_at`

## Indexing Rules

Index creation runs in a background task from `DysonNetwork.Sphere/Post/PostService.cs`.

### A post is indexed when

- the post is published now
- the post visibility is `Public`
- the index source text is not empty

### A post is reindexed when

- the post remains published
- the post remains public
- the effective indexed content changes

Indexed content is built from:

- `title`
- `description`
- `content`

These values are joined into one text block, hashed with SHA-256, and compared against the previous `source_hash`.

### Visibility changes

If a post later changes away from `Public`, the old index is intentionally kept.

Current behavior:

- do not delete the previous index row
- do not reindex while the post is not public

This matches the requirement for a non-destructive historical index.

## Search API

The post listing endpoint now supports semantic search:

```http
GET /api/posts?query=your+query
GET /api/posts?query=your+query&searchEngine=fulltext
```

Behavior:

- no `searchEngine`: uses existing `ILIKE` full-text matching
- `searchEngine=semantic`: uses embedding search
- `searchEngine=semetric`: same as semantic

Semantic search flow:

1. Sphere sends the query text to `DyEmbeddingService`
2. Insight returns the query embedding
3. Sphere ranks `post_indices` by cosine distance
4. Sphere loads the matching posts and returns normal post payloads

If semantic embedding generation is unavailable, the endpoint currently returns `503`.

## Operational Notes

- Sphere now enables the PostgreSQL `vector` extension and uses pgvector EF support.
- The index table has a unique constraint on `post_id`.
- Indexing is best-effort and does not block post creation or update.
- Search still respects normal post visibility filtering at query time.
